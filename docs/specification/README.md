# Module Specifications

This folder contains module-level behavior baselines.

Rules:

1. Document current behavior first.
2. Use `REQ-*` and `AC-*` identifiers.
3. Keep each acceptance criterion observable and testable.
4. Mark unresolved points as:
   - `NEEDS_DECISION: ...`
   - `ASSUMPTION: ...`
5. Mark future target behavior as `PROPOSED`.

Primary cross-reference files:

- `docs/spec.md` (platform behavior catalog)
- `docs/sdd/TRACEABILITY.md` (REQ/AC/test/code mapping)
- `docs/tasks.md` (status tracker)
